1. 第一題：冒泡排序（Bubble Sort）
開發過程：
在這題中，我的目標是使用 C 語言和內聯組合語言實現冒泡排序。程序會從輸入檔案中讀取一個數字序列，然後使用冒泡排序對該序列進行排序。

理解需求： 首先，我明確了這題的要求，就是在 for 迴圈內部使用內聯組合語言來進行元素的比較和交換。

實現冒泡排序： 我使用了兩個嵌套的 for 迴圈來遍歷數組，在每次比較相鄰元素時，如果前者大於後者，就交換它們的位置。這是一個典型的冒泡排序邏輯。

使用內聯組合語言： 為了實現內聯組合語言，我將每次比較和交換的操作嵌入到 asm volatile 中。我使用了 lw 來加載數組元素，並使用 bge 進行比較。交換操作使用了 sw。

調試過程：
檢查比較邏輯： 在第一次編譯時，我發現排序結果不正確。經過檢查，發現是 bge 指令的使用不正確，應該在比較時只交換大於的情況。於是我修正了比較指令，保證了只有當 arr[j] > arr[j+1] 時才進行交換。

數組指針問題： 另外，還遇到過數組指針錯誤的問題，這是因為在內聯組合語言中需要手動管理指針，並且需要確保指向正確的內存位置。

測試過程：
我通過將測試用例提供給程式，並查看輸出結果來確認排序是否正確。
測試結果顯示，程式成功地將數組從小到大排序，符合預期。

2. 第二題：線性搜尋（Linear Search）
開發過程：
這題的目的是使用 C 語言和內聯組合語言來實現線性搜尋算法。給定一個鏈表，我需要查找目標元素，並返回其索引，如果未找到則返回 -1。

理解需求： 在這題中，我理解到我們需要遍歷鏈表，逐個比較每個節點的數據，直到找到目標元素或者遍歷整個鏈表。

實現線性搜尋： 我使用了 for 迴圈來遍歷鏈表。在每次迴圈中，我使用內聯組合語言將當前節點的數據與目標值進行比較。

使用內聯組合語言： 在組合語言部分，我使用 lw 指令來加載節點的數據，並使用 beq 來比較數據值。如果找到了目標，將當前的索引返回，否則繼續遍歷鏈表。

調試過程：
鏈表遍歷： 最開始，我的鏈表遍歷邏輯不夠清晰，導致無法正確跳轉到下一個節點。經過調試，發現問題出在指針更新部分，經過修正後，問題得到了解決。

比較邏輯： 另一个挑戰是組合語言中的比較邏輯，我需要確保正確加載和比較目標值，並返回正確的索引。最終，我確認了 beq 指令的使用，當節點的數據與目標匹配時，我成功地返回了索引。

測試過程：
測試了多組數據，包括鏈表中有目標元素和無目標元素的情況。
確認當目標存在時能正確返回索引，當目標不存在時能返回 -1，符合預期。
